# 카드 뒤집기 게임 — 사용 기술 정리

## 1. 프론트엔드

| 구분 | 기술 | 설명 |
|------|------|------|
| **마크업** | HTML5 | 단일 `index.html`, 시맨틱 구조 |
| **스타일** | CSS3 | 인라인 스타일, 미디어 쿼리 없음 (반응형 `min()`, `vw` 사용) |
| **스크립트** | Vanilla JavaScript (ES5) | `var`, IIFE, 이벤트 리스너, `fetch` API |
| **빌드/도구** | 없음 | 번들러·프레임워크·패키지 매니저 미사용 |

### CSS 사용 기능
- **Layout**: Flexbox (`display: flex`), CSS Grid (`grid-template-columns`, `repeat(4, 1fr)`)
- **카드 뒤집기**: `transform`, `rotateY`, `perspective`, `backface-visibility`, `transition`
- **디자인**: `linear-gradient`, `border-radius`, `box-sizing`

### JavaScript 사용 API
- **DOM**: `getElementById`, `querySelector`, `classList`, `innerHTML`, `addEventListener`
- **비동기**: `fetch` (Supabase REST 호출)
- **타이머**: `setInterval`, `clearInterval`, `setTimeout`, `Date.now()`

---

## 2. 백엔드 / 데이터

| 구분 | 기술 | 설명 |
|------|------|------|
| **BaaS** | [Supabase](https://supabase.com) | PostgreSQL 기반, 호스팅·DB·API 제공 |
| **연동 방식** | Supabase REST API | `fetch`로 `POST`(기록 저장), `GET`(순위 조회) |
| **인증** | Anon Key (JWT) | `apikey`, `Authorization: Bearer` 헤더 |

### 사용 테이블
- **`card_flip_scores`**: `id`, `moves`, `player_name`, `duration_seconds`, `created_at`
- **RLS**: 사용 (읽기/쓰기 정책으로 제어)

---

## 3. 게임 로직

- **덱**: 이모지 8쌍(16장), Fisher–Yates 셔플
- **규칙**: 4×4 그리드, 2장 선택 후 짝 맞추기, 이동 수·경과 시간 집계
- **제한**: 60초 시간 제한, 초과 시 실패 처리
- **상태**: `gameStarted`, `gameFailed`, `flipped`, `matched` 등으로 UI·입력 제어

---

## 4. 배포 / 호스팅

- **형태**: 정적 웹 (HTML + CSS + JS 한 파일)
- **추천**: Vercel, Netlify, GitHub Pages, Cloudflare Pages 등 정적 호스팅
- **환경 변수**: 없음 (Supabase URL·Anon Key는 코드 내 상수)

---

## 5. 개발 시 참고 도구 (선택)

- **Sequential Thinking (MCP)**: 기획·설계 단계에서 단계별 사고 정리
- **Supabase MCP**: 프로젝트·테이블 확인, SQL 실행, 키 조회 등

|------|------|
